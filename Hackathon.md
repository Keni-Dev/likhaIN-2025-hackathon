## JUDGING CRITERIA BREAKDOWN & HOW TO WIN

### Innovation and Creativity (30%) - HIGHEST WEIGHT

**How to score high:**
- Don't just digitize an existing process
- Add AI/ML elements if possible (even simple ones)
- Use creative data visualization
- Implement unique user interactions
- Think: "What hasn't been done before?"

**Coding Tips:**
- Integrate GPT API for smart features
- Use interesting sensors/IoT (if applicable)
- Creative use of existing tech

### Relevance and Impact (25%)

**How to score high:**
- Show clear understanding of the problem
- Present real data/statistics about the problem
- Demonstrate tangible benefits
- Quantify impact: "Save X hours/week" or "Reduce costs by Y%"

**Presentation Tips:**
- Start with a compelling story
- Use local Bulacan examples
- Show before/after scenarios

### Feasibility (20%)

**How to score high:**
- Actually have a working demo (even if basic)
- Realistic technology choices
- Clear implementation roadmap
- Show you thought about real-world deployment

**Coding Tips:**
- Focus on core features that WORK
- Better to have 3 polished features than 10 broken ones
- Test your demo multiple times

### Clarity and Organization (15%)

**How to score high:**
- Clean, logical presentation structure
- Clear speaking without jargon
- Well-organized code (judges might look)
- Professional pitch deck design

**Presentation Tips:**
- Use the pitch deck template from the workshop
- Practice transition between slides
- Time your presentation (aim for 8-10 minutes)

### Scalability and Sustainability (10%)

**How to score high:**
- Show business model (even simple)
- Discuss future features
- Explain how it could grow
- Address maintenance/updates

**Presentation Tips:**
- Have a "roadmap" slide
- Mention partnerships or revenue model
- Show market potential

### 1:00 PM - 4:00 PM: FINAL PITCHING AND DEMO

**Presentation Structure (10 minutes max):**

1. **Hook (30 seconds)**
   - Compelling problem statement
   - Personal story or striking statistic

2. **Problem Deep-Dive (1.5 minutes)**
   - Who faces this problem?
   - Current solutions and their limitations
   - Why this matters for Smart Cities

3. **Solution Overview (2 minutes)**
   - Your innovation
   - Key features
   - How it works (high-level)

4. **Live Demo (4 minutes)**
   - Show 3-4 key features
   - Walk through user journey
   - Highlight the "wow" moment

5. **Impact & Business Model (1.5 minutes)**
   - Metrics: time saved, cost reduced, lives improved
   - Scalability
   - Sustainability/revenue model

6. **Conclusion & Call-to-Action (30 seconds)**
   - Summary of benefits
   - Future vision
   - Thank judges

**Demo Tips:**
- Use fake but realistic data
- Have a backup video if live demo fails
- Practice the demo 10+ times
- Have screenshots ready
- Start demo from a clean state

**Q&A Preparation:**
Common questions:
- "How is this different from [existing solution]?"
- "What challenges did you face?"
- "How would you handle [technical concern]?"
- "What's your go-to-market strategy?"
- "How much would this cost to implement?"
